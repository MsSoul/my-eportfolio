CETE QUIZ GAME

Full-Stack Development, Event Deployment & Department Showcase

The CETE Quiz Game is an interactive, full-stack web application designed and engineered to gamify learning and introduce participants to the fundamentals of modern web development. Originally conceptualized and deployed as a competitive event attraction, the platform balances high-performance real-time scoring mechanics with an accessible UI/UX.

Built with a robust architectural backend, the system handles multi-user database transactions simultaneously. Following its success in internal engagement, the web application was selected by administration to serve as a premier technical demonstration representing the capabilities and engineering standards of the Computer Science and IT department.

Key Event Milestones
  • College Intramurals 2024 Launch: Originally built and deployed for our College Intramurals event. It successfully served as an interactive tournament module to introduce students and participants to the core logic behind dynamic web application development.
  • Departmental Open House Showcase 2024: Selected as a centerpiece display for the school-wide Open House. Visiting students from outside schools interacted firsthand with the live system, using the game mechanics to learn how web servers, reactive designs, and backend databases connect seamlessly in real-world scenarios.
Mechanics & Gameplay Flow

The game operates on a dynamic, fast-paced tournament loop designed to challenge participants while ensuring a unique experience for every player:

  1. Dynamic Question Selection & Backend Shuffling: Players select a number from 1 to 50. To prevent predictability, the system maintains a pool of 50 questions that are automatically shuffled in the backend. Even if consecutive players select the same number, a fresh, non-repeating question is dynamically served.
  2. The 20-Second Countdown Challenge: Once a number is chosen, the system displays the question and initiates a strict 20-second countdown timer, testing the player's quick-thinking and web development knowledge under pressure.
  3. Automated Revelation & Facilitator Validation: Upon expiration of the 20-second timer, the correct answer is instantly revealed on-screen. The event facilitator then physically confirms and records whether the player won or lost the round.
  4. Victory Rewards & Leaderboard Tracking: If the player answers correctly, the system prompts them to log their details (Name, Course, and Year Level) into the tournament database, unlocking an interactive "Mystery Gift" selection module.
  5. Instant Replay Loop: If a player loses, the system resets the session state immediately, allowing them to jump right back into the queue and try their luck with a new shuffled question pool.
Technology Used
Frontend UI/UX: HTML5, Custom CSS3 Styling
Backend Environment: Node.js Framework Layer
Database Management: MongoDB Storage Tier
System Architecture: Model-View-Controller (MVC) Concept
Core Engineering: Session Management, RESTful Routing, Secure DB Queries
Testing & Quality Assurance
Manual Testing: Performed end-to-end testing of all user workflows, including question selection, countdown timer, answer reveal, winner registration, and replay functionality.
Functional Testing: Verified question generation, countdown timer, leaderboard registration, mystery gift selection, and tournament flow behaved according to system requirements.
Validation Testing: Confirmed that the application met the intended event requirements and provided a complete, reliable quiz experience for participants.
Regression Testing: Re-tested existing features after implementing enhancements and bug fixes to ensure stable system behavior throughout the event.
Black Box Testing: Evaluated the system through user interactions without referencing the underlying implementation, ensuring correct outputs for various inputs.
White Box Testing: Verified backend logic including question shuffling, session handling, routing, database operations, and game flow during development.
Database Testing: Verified participant records, winner information, leaderboard entries, and question data were accurately stored and retrieved from MongoDB.
Session Testing: Validated session resets, replay functionality, timer expiration, and prevention of duplicate gameplay issues.
Compatibility Testing: Tested the web application across multiple browsers and screen resolutions to ensure consistent functionality during live demonstrations.
Usability Testing: Evaluated navigation, readability, responsiveness, and participant interaction to provide an intuitive experience during school events.
Edge Case Testing: Tested duplicate participant entries, invalid inputs, timer expiration, repeated question selection, and unexpected user actions.
