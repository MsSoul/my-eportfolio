TIC-TAC-TOE MOBILE GAME

Mobile Application Development & Academic Exhibition Live-Demo

The Tic-Tac-Toe Mobile Application is a cross-platform mobile game engineered as an educational model and interactive deployment using Flutter and Dart. Developed to demonstrate rapid state management and reactive widget rendering, the app translates classic algorithmic turn logic into a fluid user experience.

During the HTC Open House Event, this application was featured alongside full-stack web platforms to provide an end-to-end departmental showcase. The primary objective was to demystify software engineering for campus visitors, demonstrating precisely what happens behind the screen, how standalone components handle device computations, and how mobile application runtimes handle input parameters relative to traditional web systems.

Open House Interactive Showcasing
  • Behind-The-System Code Demos: Used as a live visual reference tool to display source code structures to prospective students. Visitors observed how Dart logic and Flutter's widget trees render into operational interface buttons and visual UI themes in real time.
  • Web vs. Mobile Paradigms: Deployed as a comparative module to educate visitors on the operational differences between dynamic multi-tier websites (like the CETE Quiz Game) and client-focused mobile application processes.
Application Mechanics & Gameplay Flow

The game loop is structured to guide users through a smooth, intuitive mobile experience from setup to victory:

  1. Landing Page Token Selection: Upon launching the application, users arrive at a landing screen where the two players custom-configure their match setup. Here, players choose their preferred game tiles (assigning who plays as 'X' and who plays as 'O') before initiating the session.
  2. Turn-Based Grid Gameplay: Once the match starts, players are routed to a clean, interactive 3x3 grid. The app dynamically tracks and switches player turns sequentially after each tap, updating the tapped cell instantly with the active player's chosen tile while locking out previously filled spaces.
  3. Win Condition & Match Reset: The system continuously scans the grid layout after every move. As soon as a matching line of three tiles is achieved horizontally, vertically, or diagonally—or if all spaces fill up in a draw—the system freezes inputs, highlights the match outcome, and provides a single-tap restart feature to reset the board back to its baseline setup.

Code compilation and layout prototyping were handled inside VS Code, utilizing GitHub for secure codebase version tracking. Development optimizations, logical validations, and error-handling routines were accelerated leveraging AI assistance via ChatGPT and Gemini models.

Testing & Quality Assurance
Manual Testing: Performed end-to-end testing of gameplay, including player setup, turn switching, win detection, draw scenarios, and game reset functionality.
Functional Testing: Verified player token selection, board interactions, turn management, winner detection, draw conditions, and restart functionality.
Validation Testing: Confirmed that the application met the intended gameplay requirements and delivered a complete, responsive mobile gaming experience.
Regression Testing: Re-tested existing game features after implementing logic improvements and bug fixes to ensure stable gameplay.
Black Box Testing: Evaluated gameplay through user interactions by verifying expected outcomes without referencing the application's internal implementation.
White Box Testing: Verified game logic, state management, win-condition algorithms, input validation, and widget rendering during development.
UI Testing: Verified responsive layouts, button interactions, visual feedback, and consistent interface behavior across application screens.
Edge Case Testing: Tested invalid taps, repeated cell selections, simultaneous winning conditions, draw scenarios, and rapid user interactions.
Device Compatibility Testing: Validated application behavior across multiple Android devices, screen sizes, and orientations to ensure a consistent user experience.
Usability Testing: Evaluated navigation, gameplay flow, visual clarity, and overall ease of use to provide an intuitive experience for players.
