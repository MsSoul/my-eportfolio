PROPERTY PLANT AND EQUIPMENT TRACKING SYSTEM (PPETS)

Feb to May 2026 - City Accountant's Office

The Property Plant and Equipment Tracking System (PPETS) formerly known as ADTS (Asset Distribution and Tracking System) is an on-the-job training project that I developed during my Computer Science Internship at the City Accountant's Office of LGU General Santos City. The system was designed to help improve and organize the tracking and management of government-owned assets through a more efficient digital workflow.

I built the mobile application using Flutter and Dart, while the backend was developed with Node.js and Express. The database structure and asset records were managed using MySQL Workbench. The system includes features for asset monitoring, borrowing and returning transactions, transfer requests, notifications, and record management.

During development, I used VS Code for coding and GitHub for version control and project management. I also used AI tools such as Gemini and ChatGPT to assist with debugging, improving workflows, and refining system features. The final system helped reduce manual processes and provided a more organized and trackable asset management solution for the office.

Technology Used
Frontend: Flutter, Dart
Backend: Node.js, Express.js
Database: MySQL Workbench
Hosting: Local Server Deployment
Tools: VS Code, GitHub, Gemini, ChatGPT
Testing & Quality Assurance
Manual Testing: Performed end-to-end testing of core asset management workflows, including asset registration, borrowing, returning, transfers, notifications, and user management.
Functional Testing: Verified that all system features operated according to the defined business requirements and expected user workflows.
Validation Testing: Confirmed that the system satisfied the operational requirements of the City Accountant's Office for efficient asset tracking and record management.
Regression Testing: Re-tested existing modules after implementing enhancements and bug fixes to ensure previously working functionality remained stable.
Black Box Testing: Evaluated system functionality through user interactions by verifying expected outputs for various inputs without considering internal source code.
White Box Testing: Verified backend logic, REST API workflows, validation rules, database queries, and application processing during development.
API Testing: Tested REST API endpoints to validate request handling, authentication, response accuracy, and synchronization between the mobile application and backend services.
Database Testing: Verified CRUD operations, asset records, borrowing transactions, transfer history, and relational data integrity within MySQL.
Integration Testing: Validated communication between the Flutter mobile application, Node.js backend, and MySQL database throughout the asset management workflow.
Edge Case Testing: Tested invalid asset requests, duplicate records, incomplete transactions, unauthorized access attempts, and unexpected user inputs.
Device Compatibility Testing: Verified consistent application behavior across multiple Android devices and different screen sizes.
Usability Testing: Evaluated navigation, interface responsiveness, and workflow efficiency to ensure the application was practical for daily office operations.
